Comprehensive Guide: Building Pong in Godot 4 with GDScript

Introduction

In this tutorial, we will build Pong from scratch using Godot 4 and GDScript, without using the Godot Editor for scene creation. We will write everything in code inside VS Code. This guide is aimed at complete beginners, so we will introduce basic programming concepts, math principles, and step-by-step explanations for every part.

By the end of this tutorial, you will have a fully functional Pong game and an understanding of physics, object-oriented programming (OOP), and collision handling in Godot.

Project Overview

Our game will consist of three main components, written in separate scripts:

  1. Paddle (paddle.gd) – Handles both player and AI movement.
  2. Ball (ball.gd) – Controls movement, collisions, and scoring.
  3. GameManager (main.gd) – Manages game setup and rules.

Everything will be instantiated in code—there will be no manually created .tscn files except for the main.tscn, which serves as the game’s entry point.

Step 1: Creating the Paddle (paddle.gd) – A Beginner-Friendly Breakdown

In this step, we will create the paddle for our Pong game. The paddle is a rectangular object that moves up and down, controlled either by the player or an AI (computer opponent). Since we are writing everything in GDScript within VS Code, this guide will explain every concept in detail so that even someone new to programming can understand what’s happening.

📌 1.1: What is a Paddle in Pong?

Before writing code, let’s break down what the paddle does:

  • It moves up and down when controlled by the player.
  • It can also be controlled by AI, which will react to the ball’s movement.
  • It has a physical presence (so the ball bounces off it).

📌 1.6: Preventing the Paddle from Passing Through Walls (Collisions)

Right now, our paddle has no physical properties—it can move through anything. We need to add a collision shape to stop the ball from passing through it.

✍️ Adding a Collision Box

var collider = CollisionShape2D.new()
var shape = RectangleShape2D.new()
shape.size = Vector2(WIDTH, HEIGHT)
collider.shape = shape
add_child(collider)

Understanding CollisionShape2D in Godot (1.6 Expanded Explanation)

In 1.6: Preventing the Paddle from Passing Through Walls (Collisions), we introduced the following code snippet:

var collider = CollisionShape2D.new()
var shape = RectangleShape2D.new()
shape.size = Vector2(WIDTH, HEIGHT)
collider.shape = shape
add_child(collider)

At a high level, this creates a collision box for the paddle. However, this process involves multiple steps that must be understood individually.

What is CollisionShape2D?

CollisionShape2D is an abstract container for a physical shape in a 2D game.

  • It does not do anything on its own.
  • It cannot detect collisions until it has a defined shape inside it.
  • It serves as an interface between physics objects (like paddles) and the Godot physics engine.

Why Does CollisionShape2D Need a Shape?

A CollisionShape2D is an empty object until we assign it a specific shape (such as a rectangle, circle, or polygon). This shape determines how the object interacts with other objects in the game world.

Godot requires a shape because:

  1. Shapes define the object’s physical boundaries—Without a shape, there is no way to determine where the paddle “begins” and “ends.”
  2. The physics engine needs a defined space to calculate collisions—Without a shape, the engine does not know where to apply collision responses.
  3. Different shapes provide different collision behaviors—A rectangle is good for walls and paddles, a circle is good for balls, and a polygon allows custom shapes.

This means that simply adding a CollisionShape2D to an object does not create collision detection. We must define a shape and assign it to the CollisionShape2D for it to work.

Step-by-Step Breakdown of Collision Setup

Let’s examine each line of code and explain why it is necessary.

var collider = CollisionShape2D.new()
  • Creates a new CollisionShape2D object.
  • This is an empty collision object—it has no defined size or shape yet.
var shape = RectangleShape2D.new()
  • Creates a rectangle shape (a subclass of Shape2D).
  • This is a physical definition of what the paddle should look like to the physics engine.
shape.size = Vector2(WIDTH, HEIGHT)
  • Defines the size of the shape based on the paddle’s constants (WIDTH and HEIGHT).
  • This ensures that the collision box matches the paddle’s visual representation.
collider.shape = shape
  • Assigns the shape to the collider so it has a physical presence.
  • Now, the physics engine recognizes this as a solid object.
add_child(collider)
  • Attaches the collider to the paddle so that it moves with the paddle.
  • Without this, the paddle would move, but its collision box would remain in place.

1.7: Moving the Paddle (Player Input & AI Logic)

At this point, our paddle exists, has a visual representation, and has a collision shape that prevents other objects from passing through it. However, the paddle does not move yet. In this step, we will implement player-controlled movement and AI-controlled movement, while explaining new concepts like inherited physics properties (velocity.y), movement handling (move_and_collide()), and basic math functions (sign()).

📌 1.7.1: How Do We Move the Paddle?

In Godot, movement in a 2D game is controlled by modifying an object’s position over time. Because our paddle extends CharacterBody2D, it automatically inherits some useful built-in physics properties.

One of these properties is:

velocity: Vector2

This represents the speed and direction of movement for a CharacterBody2D object.

  • velocity.x → Controls horizontal movement (left/right).
  • velocity.y → Controls vertical movement (up/down).

Since our paddle only moves up and down, we will modify velocity.y to determine the paddle’s movement speed.

📌 1.7.2: Understanding move_and_collide()

Godot provides a built-in function for moving physics objects:

move_and_collide(velocity * delta)

This function moves the object by the specified velocity and checks for collisions along the way.

  • If no collision occurs, the object moves freely.
  • If a collision occurs, the object stops at the collision point, preventing it from passing through other objects.

This function is essential for ensuring that the paddle stops when it reaches a wall instead of continuing through it.

📌 1.7.3: Implementing Player-Controlled Movement

We want the player to move the paddle up and down using the keyboard. In Godot, we retrieve player input using:

Input.get_axis("ui_up", "ui_down")
  • Returns 1 if “ui_up” is pressed (moving up).
  • Returns -1 if “ui_down” is pressed (moving down).
  • Returns 0 if neither key is pressed (no movement).

Adding Player Movement to paddle.gd

func _player_move(delta):
    velocity.y = Input.get_axis("ui_up", "ui_down") * SPEED
    move_and_collide(velocity * delta)

Breaking It Down

  1. velocity.y = Input.get_axis("ui_up", "ui_down") * SPEED

    • If the player presses “ui_up”, velocity becomes negative, moving the paddle up.
    • If the player presses “ui_down”, velocity becomes positive, moving the paddle down.
    • If no keys are pressed, velocity is zero, and the paddle does not move.
    • Multiplying by SPEED ensures the paddle moves at a consistent rate.

  2. move_and_collide(velocity * delta)

    • Moves the paddle based on velocity.
    • delta scales the movement to match the game’s frame rate, ensuring smooth movement.

At this point, the player can now control the paddle using the keyboard.

📌 1.7.4: Implementing AI-Controlled Movement

The AI paddle needs to track the ball’s Y position and move accordingly. To ensure fairness, the AI should have a reaction delay so it does not immediately respond to every ball movement.

Introducing sign()

The AI will use the sign() function:

sign(value)
  • Returns 1 if value is positive.
  • Returns -1 if value is negative.
  • Returns 0 if value is zero.

This function helps determine which direction the AI should move.
For example:

sign(ball.position.y - position.y)
  • If the ball is above the paddle → Returns -1, meaning “move up”.
  • If the ball is below the paddle → Returns 1, meaning “move down”.
  • If the ball is at the same height → Returns 0, meaning “do nothing”.

Adding AI Movement to paddle.gd

var ai_reaction_delay := 0.1
var _reaction_timer := 0.0

func _ai_move(delta):
    _reaction_timer += delta
    if _reaction_timer >= ai_reaction_delay:
        var ball = get_tree().get_first_node_in_group("ball")
        if ball:
            velocity.y = sign(ball.position.y - position.y) * SPEED * 0.8
        _reaction_timer = 0
    move_and_collide(velocity * delta)

Breaking It Down

  1. _reaction_timer += delta

    • Increases the timer based on the time that has passed since the last frame.
    • This ensures the AI only updates its movement at fixed intervals, making it more natural.

  2. if _reaction_timer >= ai_reaction_delay:

    • Only moves the AI after a minimum delay, preventing it from reacting too quickly.

  3. var ball = get_tree().get_first_node_in_group("ball")

    • Finds the ball in the game and stores it in a variable.

  4. velocity.y = sign(ball.position.y - position.y) * SPEED * 0.8

    • Uses sign() to move the AI toward the ball.
    • Multiplies by SPEED * 0.8 so the AI moves slightly slower than the player.

Understanding sign() and How It Helps AI Movement

In this section, we introduce sign(), a mathematical function that helps determine direction. However, we have not yet explained why it works or how to think about it in other situations. This section will break down the math behind sign(), explain how it conceptualizes movement, and compare it to alternative approaches.

📌 What Does sign(value) Do?

The sign() function is a basic mathematical function that returns:

Input Value Output of sign(value) Meaning
value > 0 1 Positive direction (move up/down/right)
value < 0 -1 Negative direction (move down/up/left)
value == 0 0 No movement

Instead of checking if a value is greater than or less than another value manually, sign() tells us which direction the difference is pointing.

📌 How Do We Use sign() to Move the AI Paddle?

In our AI movement function, we determine whether the paddle should move up or down using:

velocity.y = sign(ball.position.y - position.y) * SPEED * 0.8

Let’s break this expression into parts:

  1. ball.position.y - position.y

    • This finds the vertical distance between the ball and the paddle.
    • If the result is positive, the ball is below the paddle.
    • If the result is negative, the ball is above the paddle.

  2. sign(ball.position.y - position.y)

    • This converts the difference into a directional value (1, -1, or 0).
    • If the ball is below the paddle, it returns 1 (move down).
    • If the ball is above the paddle, it returns -1 (move up).
    • If the ball is perfectly aligned, it returns 0 (stay still).

  3. * SPEED * 0.8

    • This scales the movement so that the paddle moves at 80% of the player’s speed.

Thus, velocity.y = sign(ball.position.y - position.y) * SPEED * 0.8 allows the AI to automatically adjust its direction without any if-statements.

📌 Alternative Approach: Using If-Statements Instead of sign()

If we did not use sign(), we would have to write manual conditions like this:

if ball.position.y > position.y:
    velocity.y = SPEED * 0.8  # Move down
elif ball.position.y < position.y:
    velocity.y = -SPEED * 0.8  # Move up
else:
    velocity.y = 0  # Stay still

This is a valid approach but has downsides:

  • Requires multiple lines of code.
  • Harder to read and modify later.
  • More difficult to use in other scenarios (requires additional logic for left/right movement).

Using sign() makes the logic simpler, more scalable, and easier to reuse.

📌 Conceptualizing sign() for Future Use

Whenever you need to determine direction, consider using sign().

Examples of When to Use sign():

  1. Moving toward a target position

    var direction = sign(target_position - current_position)
velocity = direction * speed

  2. Checking if an object is to the left or right of another object

    var x_direction = sign(object2.x - object1.x)  # -1 if left, 1 if right

  3. Applying a force in the correct direction

    apply_impulse(Vector2(sign(player_position.x - enemy_position.x) * force, 0))

Using sign() simplifies many movement and AI behaviors where direction needs to be determined dynamically.

  1. _reaction_timer = 0

    • Resets the timer so the AI will wait before updating its movement again.

  2. move_and_collide(velocity * delta)

    • Moves the AI paddle and stops it if it collides with something.

At this point, the AI paddle is functional and follows the ball with a natural reaction delay.

📌 2.1: What is the Ball’s Role in Pong?

Before we write any code, let’s understand what the ball needs to do:

We will implement this behavior using GDScript and Godot’s physics system.

📌 2.6: Implementing Ball Movement and Bouncing

Now we’ll make the ball move and handle collisions with paddles and walls.

✍️ Adding Movement Logic

func _physics_process(delta):
    var collision = move_and_collide(velocity * delta)

    # Check paddle collisions
    if collision:
        velocity = velocity.bounce(collision.get_normal())

    # Manually handle viewport bouncing if walls are not being detected
    if position.y <= 0:  # Top of screen
        velocity.y = abs(velocity.y)  # Force positive y-velocity
    elif position.y >= viewport.y:  # Bottom of screen
        velocity.y = -abs(velocity.y)  # Force negative y-velocity
    
    _check_scoring()

🔹 What’s Happening Here?

  • _physics_process(delta) runs every frame to update physics
  • Paddle collisions use Godot’s built-in collision system
  • Wall collisions are handled manually by checking screen boundaries
  • Score checking happens after movement is processed

📌 2.7: Resetting the Ball

When a point is scored, we need to reset the ball to the center with a random direction.

✍️ Implementing Reset Logic

func reset():
    var x_direction = [-1, 1][randi() % 2]  # Randomly choose -1 (left) or 1 (right)
    var y_direction = randf_range(-1.0, 1.0)  # Ensure a wider range of angles

    velocity = Vector2(x_direction, y_direction).normalized() * initial_speed
    position = get_viewport_rect().size / 2

🔹 What’s Happening Here?

  • Random horizontal direction using array selection
  • Random vertical angle using randf_range
  • Normalized velocity ensures consistent speed
  • Centers the ball on the screen

📌 2.8: Handling Scoring

Finally, we need to detect when a player scores and update the game accordingly.

✍️ Adding Score Detection

func _check_scoring():
    viewport = get_viewport_rect().size
    if position.x < 0:
        get_parent().player2_score += 1
        reset()
    elif position.x > viewport.x:
        get_parent().player1_score += 1
        reset()

🔹 What’s Happening Here?

  • Updates viewport size (in case of window resizing)
  • Checks if ball goes past left or right boundaries
  • Updates appropriate player’s score
  • Calls reset() to start new round

📌 2.9: Initializing the Ball

Don’t forget to add the _ready function to initialize the ball when it’s added to the scene!

func _ready():
    reset()

This ensures the ball starts moving as soon as the game begins.

Step 3: Creating the Game Manager (main.gd)

With the Paddle and Ball classes completed, we now need a central system to manage the game. The Game Manager is responsible for creating and positioning the game objects, keeping track of scores, and ensuring the game runs correctly.

This step will focus on object instantiation, score tracking, and viewport-aware positioning so that the paddles and ball appear in the correct locations.

3.1: The Role of the Game Manager

The Game Manager will:

  2. Create the ball – It will start in the center of the screen.
  3. Track the score – It will update when a player scores a point.
  4. Position all objects correctly based on the screen size.

Unlike the Paddle and Ball classes, which represent individual objects, the Game Manager is not a physical object in the game. It is a controller that manages the creation and behavior of other objects.

At this stage, the Game Manager exists but does nothing. Next, we will define game variables.

3.5: Creating the Ball

Defining a Ball Variable

var ball

Instantiating the Ball
